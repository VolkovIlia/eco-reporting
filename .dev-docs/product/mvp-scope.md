# ЭкоОтчёт -- MVP Scope

**Продукт:** ЭкоОтчёт (eco-reporting)
**Платформа:** Комплаенс.РФ
**Дата:** 13 марта 2026
**Версия:** 0.1.0-mvp

---

## 1. Гипотеза продукта

**Для** экологов и руководителей производственных предприятий, обязанных сдавать экологическую отчётность в Росприроднадзор,
**существует потребность в** SaaS-инструменте для автоматизированного заполнения, валидации и генерации форм 2-ТП и ПЭК,
**потому что** текущие альтернативы -- консалтинг за 100-500 тыс. руб./отчёт или бесплатный, но примитивный "Модуль природопользователя" -- слишком дороги или неудобны;
**это приведёт к** привлечению 100+ платящих клиентов за первые 6 месяцев и MRR 500 тыс. руб. к месяцу 6.

**Уровень доказательности:** D (предположение, требует валидации через интервью)

**Ключевые предположения:**
- A-01: Экологи предприятий готовы платить 5 000 руб./мес. за SaaS вместо консалтинга [D]
- A-02: TAM = ~50 000 компаний, обязанных сдавать экоотчётность [B -- реестр Росприроднадзора]
- A-03: 1С НЕ выпустит модуль экологической отчётности в ближайшие 12 месяцев [C]
- A-04: Приказ Минприроды #757 от 26.12.2024 создал волну переучивания, повышающую спрос [B]
- A-05: Формы 2-ТП достаточно стандартизированы для автоматизации командой из 2-3 человек [C]

**Критерии успеха (3 месяца):**
- 30+ платящих клиентов
- ARPU >= 5 000 руб./мес.
- NPS >= 30
- Monthly churn < 3%
- P95 latency API < 500ms

## 2. Целевой пользователь

### Персона 1: Штатный эколог (основная)

**Имя:** Марина, 35 лет
**Должность:** Инженер-эколог на производственном предприятии (завод, пищевое производство, склад)
**Компания:** 50-500 сотрудников, 1-3 производственные площадки

**Боли:**
- Каждый январь-март -- "экологический ад": 4-6 отчётов с жёсткими дедлайнами
- Приказ #757 полностью изменил формы -- нужно переучиваться
- "Модуль природопользователя" -- десктопное ПО без валидации, зависает, теряет данные
- Ставки экосбора растут ежегодно (+15% в 2025), ошибка в расчёте = переплата или штраф
- Нулевые декларации НВОС теперь обязательны -- нельзя просто пропустить

**Задачи (JTBD):**
- "Когда наступает дедлайн 2-ТП, я хочу быстро заполнить форму с проверкой ошибок, чтобы сдать отчёт вовремя и без штрафов"
- "Когда выходит новый приказ Минприроды, я хочу понять, что изменилось в формах, чтобы не переделывать отчёт"

**Текущие альтернативы:**
1. Консалтинг (100-500K руб./отчёт) -- дорого для среднего бизнеса
2. "Модуль природопользователя" -- бесплатный, но ужасный UX, десктоп, без валидации
3. Excel-шаблоны (ЭКО-Профи) -- бесплатно, но без автоматизации
4. Штатный эколог заполняет вручную -- медленно, ошибки

### Персона 2: Руководитель / бухгалтер малого предприятия (вторичная)

**Имя:** Алексей, 45 лет
**Должность:** Директор малого производственного предприятия (10-50 человек)
**Контекст:** Нет штатного эколога, экоотчётность -- побочная задача

**Боли:**
- Не понимает, какие отчёты обязан сдавать
- Штраф за несдачу 2-ТП: 20 000-80 000 руб.
- Консалтинг слишком дорог для малого бизнеса

## 3. MVP-скоуп (User Stories)

### MUST HAVE (без этого продукт бессмыслен)

#### US-001: Регистрация и авторизация
**Как** пользователь, **я хочу** зарегистрироваться по email и войти в систему, **чтобы** безопасно работать с данными моего предприятия.

**Приоритет:** RICE = (500 x 3 x 100%) / 1 = 1500 | MUST

**Критерии приёмки:**
```gherkin
Given пользователь заполнил форму регистрации с валидным email и паролем >= 8 символов
When пользователь нажимает "Зарегистрироваться"
Then создаётся аккаунт и отправляется письмо подтверждения

Given пользователь вводит существующий email при регистрации
When пользователь нажимает "Зарегистрироваться"
Then отображается ошибка "Этот email уже зарегистрирован"

Given пользователь ввёл корректные email и пароль
When пользователь нажимает "Войти"
Then пользователь перенаправляется на дашборд с JWT-токеном (TTL 24h)
```

**Экран:** Стандартная форма регистрации/входа. Поля: email, пароль, название организации, ИНН (опционально). Кнопка "Войти через Госуслуги" -- заглушка для v2.

---

#### US-002: Управление объектами (площадками)
**Как** эколог, **я хочу** добавить производственные площадки моего предприятия, **чтобы** привязать к ним экологическую отчётность.

**Приоритет:** RICE = (500 x 3 x 90%) / 1.5 = 900 | MUST

**Критерии приёмки:**
```gherkin
Given пользователь авторизован и находится на странице "Объекты"
When пользователь нажимает "Добавить объект" и заполняет: название, адрес, категория НВОС (I-IV), вид деятельности (ОКВЭД)
Then объект сохраняется и появляется в списке

Given объект имеет категорию НВОС I или II
When пользователь открывает объект
Then система показывает список обязательных отчётов для этой категории (2-ТП воздух, 2-ТП вода, 2-ТП отходы, ПЭК, НВОС)

Given объект имеет категорию НВОС III
When пользователь открывает объект
Then система показывает сокращённый список обязательных отчётов (2-ТП отходы, декларация НВОС)
```

**Экран:** Список объектов в виде карточек. Каждая карточка: название, адрес, категория НВОС (цветной бейдж I/II/III/IV), количество привязанных отчётов, статус ("Все сданы" / "2 отчёта просрочены"). Кнопка "+ Добавить объект" открывает модальное окно с формой.

---

#### US-003: Заполнение формы 2-ТП (отходы)
**Как** эколог, **я хочу** заполнить форму 2-ТП (отходы) с подсказками и автоматической валидацией, **чтобы** сдать корректный отчёт в Росприроднадзор.

**Приоритет:** RICE = (500 x 3 x 95%) / 3 = 475 | MUST

**Критерии приёмки:**
```gherkin
Given пользователь выбрал объект и нажал "Создать отчёт 2-ТП (отходы)"
When открывается форма
Then форма соответствует актуальному формату Приказа Минприроды #757 от 26.12.2024 и содержит все обязательные разделы

Given пользователь заполняет поле "Код отхода по ФККО"
When пользователь начинает вводить код или название
Then система предлагает варианты из справочника ФККО с автодополнением

Given пользователь ввёл данные по объёмам образования отходов
When пользователь нажимает "Проверить"
Then система валидирует: баланс (образовано = использовано + обезврежено + размещено + передано +/- остаток), коды ФККО существуют, обязательные поля заполнены

Given валидация нашла ошибки
When пользователь смотрит результат проверки
Then ошибки подсвечены красным с объяснением ("Баланс не сходится: разница 2.5 тонн в строке 3")

Given форма заполнена и валидация пройдена
When пользователь нажимает "Сохранить черновик"
Then данные сохраняются, статус отчёта = "Черновик"
```

**Экран:** Многошаговая форма (wizard). Шаг 1: Общие сведения (отчётный период, организация, объект). Шаг 2: Таблица отходов (строки: код ФККО, наименование, класс опасности, образовано, использовано, обезврежено, размещено, передано другим, получено от других). Шаг 3: Сведения об объектах размещения. Шаг 4: Проверка и итог. Справа -- панель валидации с иконками (зелёная галочка / красный крестик).

---

#### US-004: Заполнение формы 2-ТП (воздух)
**Как** эколог, **я хочу** заполнить форму 2-ТП (воздух) с подсказками, **чтобы** отчитаться о выбросах загрязняющих веществ.

**Приоритет:** RICE = (400 x 3 x 90%) / 3 = 360 | MUST

**Критерии приёмки:**
```gherkin
Given пользователь создаёт отчёт 2-ТП (воздух)
When открывается форма
Then форма соответствует формату Приказа #757 и содержит разделы: источники выбросов, загрязняющие вещества, объёмы, мероприятия по снижению

Given пользователь заполняет поле "Загрязняющее вещество"
When пользователь начинает вводить название или код
Then система предлагает варианты из справочника загрязняющих веществ с ПДК

Given пользователь ввёл объёмы выбросов
When пользователь нажимает "Проверить"
Then система валидирует: объёмы > 0, вещества из справочника, обязательные поля заполнены, суммы по источникам = итого
```

**Экран:** Аналогичный wizard, как US-003. Таблица: источник выброса, код вещества, наименование, разрешённый объём (т/год), фактический объём (т/год). Визуальный индикатор превышения (жёлтый если >80% от разрешённого, красный если >100%).

**Как** эколог, **я хочу** заполнить форму 2-ТП (водное хозяйство), **чтобы** отчитаться о водопользовании и сбросах.

**Приоритет:** RICE = (300 x 3 x 90%) / 3 = 270 | MUST

**Критерии приёмки:**
```gherkin
Given пользователь создаёт отчёт 2-ТП (водхоз)
```

#### US-006: Генерация PDF-отчёта
**Как** эколог, **я хочу** скачать заполненный отчёт в формате PDF, **чтобы** подать его в Росприроднадзор.

**Приоритет:** RICE = (500 x 3 x 100%) / 2 = 750 | MUST

**Критерии приёмки:**
```gherkin
Given отчёт полностью заполнен и валидация пройдена без ошибок
When пользователь нажимает "Скачать PDF"
Then генерируется PDF-файл в формате, принимаемом Росприроднадзором (соответствие приложениям к Приказу #757)

Given отчёт содержит ошибки валидации
When пользователь нажимает "Скачать PDF"
Then отображается предупреждение "Отчёт содержит ошибки. Скачать всё равно?" с возможностью скачать черновой вариант

Given PDF скачан
When пользователь открывает файл
Then все данные корректно отображаются, таблицы не обрезаны, шрифты читаемы, формат A4
```

**Экран:** Кнопка "Скачать PDF" на странице отчёта. Превью PDF в модальном окне перед скачиванием.

#### US-007: Календарь дедлайнов
**Как** эколог, **я хочу** видеть календарь дедлайнов экологической отчётности, **чтобы** не пропустить сроки сдачи.

**Приоритет:** RICE = (500 x 2 x 95%) / 1 = 950 | MUST

**Критерии приёмки:**
```gherkin
Given пользователь авторизован и имеет объекты
When пользователь открывает дашборд
Then отображается таймлайн с дедлайнами: 22 января (2-ТП воздух), 2 февраля (2-ТП отходы, 2-ТП вода), 25 марта (ПЭК), 10 марта (декларация НВОС), 1 апреля (расчёт экосбора)

Given до дедлайна осталось менее 30 дней
When пользователь открывает дашборд
Then дедлайн подсвечен жёлтым

Given до дедлайна осталось менее 7 дней
When пользователь открывает дашборд
Then дедлайн подсвечен красным

Given дедлайн прошёл и отчёт не отмечен как "сдан"
When пользователь открывает дашборд
Then дедлайн отмечен красным с меткой "ПРОСРОЧЕНО"
```

**Экран:** Дашборд -- главная страница после авторизации. Вверху: горизонтальный таймлайн (январь-апрель) с маркерами дедлайнов. Ниже: карточки объектов с прогрессом ("Объект X: 2/4 отчёта готовы"). Справа: уведомления.

**Как** эколог, **я хочу** иметь встроенные справочники ФККО и загрязняющих веществ, **чтобы** не искать коды вручную.

**Приоритет:** RICE = (500 x 2 x 100%) / 2 = 500 | MUST

**Критерии приёмки:**
```gherkin
Given пользователь заполняет поле с кодом ФККО
When пользователь вводит часть кода или названия отхода

Given справочник ФККО содержит >4000 записей
When пользователь ищет по названию
Then результаты появляются менее чем за 300ms (поиск по индексу)

Given справочник загрязняющих веществ
When пользователь ищет вещество по названию или CAS-номеру
Then система показывает вещество с ПДК и классом опасности
```


### WON'T HAVE (не в MVP)

- **US-W01:** Прямая подача отчёта через API Росприроднадзора (API недокументирован)
- **US-W02:** Интеграция с 1С (требует значительных ресурсов)
- **US-W03:** Мобильное приложение (веб достаточно для MVP)
- **US-W04:** Расчёт рассеивания выбросов (узкоспециализированная задача)
- **US-W05:** Генерация полного ПЭК с нуля (требует экспертизы, в MVP только отчёт по ПЭК)
- **US-W06:** Мониторинг изменений в законодательстве (контент-задача, не техническая)
- **US-W07:** White-label для консалтинговых компаний

## 7. Что НЕ входит в MVP

1. **Прямая подача в Росприроднадзор** -- API не задокументирован, требует партнёрства
2. **Интеграция с 1С** -- слишком ресурсоёмко, клиенты экспортируют/импортируют через Excel
3. **Мобильное приложение** -- экологи работают за компьютером, не в поле
4. **Расчёт рассеивания выбросов** -- отдельный нишевый продукт (конкурент Stalker/Integral)
5. **Генерация ПЭК программы с нуля** -- требует глубокой экспертизы, в MVP только отчёт
6. **Мониторинг законодательства** -- контент-задача, в MVP ручное обновление форм
7. **Онбординг-консалтинг** -- в MVP self-service + documentation
8. **Электронная подпись (ЭЦП)** -- требует интеграции с УКЭП-провайдерами
9. **Автоматический расчёт платежей НВОС** -- сложная формула с коэффициентами, COULD для v1.1
10. **Многоязычность** -- только русский язык

## 8. Риски и митигации

| ID | Риск | Вероятность | Влияние | Митигация |
|----|------|-------------|---------|-----------|
| R-01 | 1С выпускает модуль экологической отчётности | Средняя | Высокое | Набрать 100+ клиентов до возможного выхода 1С (12+ мес. запас); фокус на UX, который 1С не сможет дать |
| R-02 | Росприроднадзор улучшает свой портал | Низкая | Высокое | Гос.порталы обновляются медленно; наша ценность -- валидация + UX + уведомления, а не просто форма |
| R-03 | Приказ Минприроды снова меняет формы | Средняя | Среднее | Модульная архитектура форм: JSON-схема определяет структуру, обновление = замена схемы без переписывания кода |
| R-04 | Экологи не готовы платить за SaaS | Средняя | Критическое | Валидировать через 10-15 интервью ДО разработки; предложить pay-per-report модель как альтернативу подписке |
| R-05 | Online Ecology набирает обороты | Низкая | Среднее | Их цены скрыты и бренд неизвестен; наше преимущество -- прозрачность + UX + актуальность форм по Приказу #757 |
| R-06 | Сезонность спроса (только январь-март) | Высокая | Среднее | Расширить на ПЭК (дедлайн март), экосбор (апрель); предложить годовую подписку с дисконтом |
| R-07 | Ошибка в формах/валидации приводит к штрафу клиента | Низкая | Критическое | Дисклеймер "инструмент автозаполнения, ответственность на пользователе"; тестирование форм с реальными экологами |

Консистентно с существующим проектом microplastics-reporting:
